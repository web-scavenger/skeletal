# skeletal

Automate skeleton loading screens for React and Next.js TypeScript projects.

`skeletal` scans your codebase, crawls your running app with Playwright to capture real element geometry, and generates pixel-accurate `.skeleton.tsx` files alongside your components — no manual shimmer code required.

## Install

```sh
npm install skeletal
# or
pnpm add skeletal
```

For browser crawl (used by `skeletal analyze`):

```sh
npx playwright install chromium
```

## Quick start

```sh
npx skeletal init
```

This creates `skeletal.config.ts` and walks you through setup. Then:

```sh
# Start your dev server, then:
npx skeletal analyze

# Or skip browser crawl to generate minimal skeletons immediately:
npx skeletal analyze --no-browser
```

## How it works

skeletal detects four patterns automatically:

| Pattern | Detection | Codemod |
|---------|-----------|---------|
| **RSC** | `async` function inside `<SkeletonWrapper>` | Attaches `.skeleton` export |
| **CSR** | non-async function inside `<SkeletonWrapper>` | Attaches `.skeleton` export |
| **lazy** | `React.lazy(() => import('./X'))` | Replaces with `lazyWithSkeleton` |
| **dynamic** | `next/dynamic(() => import('./X'))` | Replaces with `dynamicWithSkeleton` |

### RSC / CSR

Wrap your component in `<SkeletonWrapper>`:

```tsx
import { SkeletonWrapper } from 'skeletal'
import { UserCard } from './UserCard'

export default function Page() {
  return (
    <SkeletonWrapper>
      <UserCard userId="u_001" />
    </SkeletonWrapper>
  )
}
```

skeletal generates `UserCard.skeleton.tsx` and attaches it at runtime:

```tsx
// auto-generated by skeletal
Object.assign(UserCard, { skeleton: UserCardSkeleton })
```

### React.lazy

```tsx
// before
const HeavyChart = React.lazy(() => import('./HeavyChart'))

// after (auto-applied by `skeletal analyze`)
import { lazyWithSkeleton } from 'skeletal'
const HeavyChart = lazyWithSkeleton(() => import('./HeavyChart'))
```

### next/dynamic

```tsx
// before
import dynamic from 'next/dynamic'
const Map = dynamic(() => import('./Map'))

// after (auto-applied by `skeletal analyze`)
import { dynamicWithSkeleton } from 'skeletal/next'
const Map = dynamicWithSkeleton(() => import('./Map'))
```

## Primitives

Use `Sk.*` components inside your skeleton files for semantic, CSS-only shimmer blocks:

```tsx
import { Sk } from 'skeletal'

export function UserCardSkeleton() {
  return (
    <div className="user-card">
      <Sk.Avatar size={64} />
      <div>
        <Sk.Heading width="60%" />
        <Sk.Text lines={2} />
      </div>
    </div>
  )
}
```

Available primitives: `Sk.Text`, `Sk.Heading`, `Sk.Avatar`, `Sk.Image`, `Sk.Button`, `Sk.Badge`, `Sk.Number`, `Sk.Icon`, `Sk.List`, `Sk.Card`

Import the default styles (once, in your root layout):

```tsx
import 'skeletal/styles.css'
```

## CLI

```
skeletal init                    Interactive setup, creates skeletal.config.ts
skeletal analyze                 Scan, crawl, generate, and wire skeletons
skeletal analyze --no-browser    Skip Playwright crawl, generate minimal skeletons
skeletal analyze --dry-run       Preview changes without writing files
skeletal check                   Assert all skeletons are up to date (for CI)
skeletal check --json            Machine-readable JSON output
skeletal watch                   Re-analyze on component file changes
skeletal preview                 Serve skeleton previews in the browser
skeletal eject <Name>            Copy a generated skeleton to your source tree
```

## Configuration

```ts
// skeletal.config.ts
import { defineConfig } from 'skeletal'

export default defineConfig({
  devServer: 'http://localhost:3000',
  routes: ['/', '/dashboard', '/profile'],
  include: ['src/**/*.tsx'],
  exclude: ['**/*.test.tsx'],
  output: 'colocated',        // or 'directory'
  // outputDir: 'src/skeletons', // when output: 'directory'
  animation: 'shimmer',       // or 'pulse' | 'none'
  radius: 6,
})
```

## Vite plugin

```ts
// vite.config.ts
import { skeletalVitePlugin } from 'skeletal/vite'

export default {
  plugins: [skeletalVitePlugin()],
}
```

## Requirements

- TypeScript `>= 5.0`
- React `>= 18`
- Node.js `>= 18`
- Playwright `>= 1.44` (optional — only needed for `skeletal analyze` with browser crawl)
